# AI 時代前 40 個月的回顧

> 一篇從個人使用經驗出發，回顧 ChatGPT 問世後近 40 個月內，AI 在寫作、程式開發、創作與工作流程上的真實改變。

## 背景

這篇文章不是在追模型排行榜，也不是在談單一新品，而是作者回頭整理自己自 2022 年底以來與 AI 相處的經驗：從第一次被 ChatGPT 震撼，到後來使用 Claude Code 實作小專案、規劃創業、反思 AI 寫作品質與內容消費體驗。

它的價值在於：很多人對 AI 的感受已經從「好神奇」進入「到底真的有多有用」的階段，這篇正是在處理那個比較成熟的問題。

## 核心重點

### 1. 最早的震撼來自「它終於不只是玩具」

作者回想 ChatGPT 剛推出時，最強烈的感受不是它多會聊天，而是它明顯超越了舊式聊天機器人。這種跨越不是小幅進步，而是讓人立刻意識到：這次一般人也會在意，AI 不再只是科技圈自嗨。

### 2. AI 寫程式很有用，但很難精確衡量到底省了多少

作者測試過讓 AI 生成小型應用，也用過 Claude Code 做更完整的互動式開發。觀察是：

- 一次性產出通常很驚豔
- 小型專案能快速得到可運作雛形
- 反覆修改時仍容易逐漸失焦
- 最終常常要自己重寫不少內容

所以真正難回答的不是「能不能用」，而是：

- 它究竟節省多少時間？
- 會不會只是讓人把 scope 擴更大？
- 增加的功能到底是價值，還是只是更多複雜度？

這是非常誠實、也很有代表性的觀察。

### 3. Claude Code 最有價值的地方，是把自然語言變成新的電腦操作介面

作者對 Claude Code 最正面的評價，不只是「會寫程式」，而是它讓使用者能自然地對電腦下達高階意圖。這種感覺不像傳統 IDE 補全，而比較像新增了一種與電腦互動的輸入模式，和鍵盤、滑鼠、terminal 並列。

這點很重要，因為它把 AI 的價值從「生成內容」轉成「協助操作系統與工作流」。

### 4. AI 也會在心理層面推你一把

作者提到自己曾用 Claude 幫忙規劃小型 IT 服務公司的啟動計畫。即使事後看起來很多內容並不神祕，但當下確實有被鼓勵、被推動開始行動。這裡也連到一個現代很常見的現象：AI 不只是工具，也會扮演某種「助推器」或「過度附和的顧問」。

### 5. 作者依然不喜歡 AI 生成內容本身

儘管 AI 在工作上有幫助，但作者對 AI 生成文章與創作內容仍然保留。主要原因：

- 文風太平、太安全
- 看起來對，但沒有個人性
- 不是自己的聲音
- 作為讀者也會產生一種 uncanny valley 式的不適感

這代表一個很關鍵的分野：
AI 作為「生產工具」可能有價值，但作為「直接替代創作者」未必令人滿意。

## 實務啟發

- 如果你在評估 AI 對開發工作的價值，不要只問「快多少」，也要問「是否改變了我對 scope 的判斷」。
- AI coding tool 最穩定的價值，可能不是完整代寫，而是降低啟動成本、減少切換上下文、幫你快速完成雛形。
- 在內容創作上，AI 也許更適合當整理工具、提案工具、陪跑工具，而不是直接代筆。
- 對很多知識工作者來說，AI 最實際的效果可能是「促進行動」，而不是「產出完美答案」。

## 我的補充

我覺得這篇最好的地方，是它沒有走極端。它既不是 AI 狂熱文，也不是「都沒用」的反彈文，而是比較接近多數實作者現在的真實狀態：

- 有些地方真的很好用
- 有些地方很容易讓人高估
- 真正的價值還在重新校準

如果要一句話濃縮，這篇其實是在說：
**AI 已經足夠重要到不能忽視，但還沒成熟到可以用單一指標判斷它的價值。**

# 當資深工程師的直覺失效時

> 一篇寫給資深 IC / Principal / 技術領導者的提醒：在 AI 時代，你最需要更新的不是 title，而是你的 heuristics。

## 背景

作者從一個很有畫面的比喻開始：很多資深工程師把自己想像成退役冠軍教練，靠過往經驗、品味、人脈與判斷帶隊。但問題是，現在比賽規則變了——我們已經不是在以前那個開發環境裡工作。

## 核心重點

### 1. 很多過去正確的 heuristics，現在已經不再可靠

作者列出一整串可能正在失效的判斷框架：

- 什麼叫 maintainable
- 寫程式、整合 library、依賴 service 的相對成本
- 什麼算是好 API
- 什麼叫真的理解程式碼
- service boundary 應該畫在哪裡
- security / data integrity 要放在哪一層
- 什麼容易、什麼困難

這不是在說經驗沒用，而是在說：**環境變了，經驗累積出的捷徑也要重新驗證。**

### 2. 技術領導現在最重要的能力，是謙虛、勇氣與好奇心

作者給出的新工作描述很漂亮：

- 有謙虛，承認很多 heuristics 已經錯了
- 有勇氣，相信不是所有東西都要丟掉
- 有好奇心，主動分辨哪些還有效、哪些該更新

這比「嘴上談趨勢」更實際，因為它要求領導者真的去碰工具、碰現場。

### 3. 更新 heuristics 的唯一方法，是重新下場做事

作者非常明確地說：

- build something real
- build prototypes
- 做很多小實驗
- 主動挑戰以前認為不可行或太貴的做法
- 找出你擔心新工具會出問題的地方，親手驗證並修補

也就是說，面對 AI 時代，最危險的技術領導不是不知道，而是**只剩評論，沒有實作**。

## 實務啟發

- 如果你是資深工程師，現在最值得投資的不是更會評論 AI，而是親手做更多小型實驗。
- 不要急著把過去的規則全部丟掉，但也不要假設它們仍然成立。
- 團隊裡最有價值的人，會是那些既有經驗、又願意主動更新判斷模型的人。

## 我的補充

我很認同這篇，因為它講的其實不是 AI，而是**專業者如何面對典範轉移**。

真正厲害的人，不是永遠都對，而是當環境變了，能最快重新校準自己。對資深工程師來說，最丟臉的不是承認自己舊直覺失效，而是明明失效了還硬撐著教別人。
